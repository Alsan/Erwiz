Ervizについての覚え書き

  • ErvisはASCII とUTF-8のみをサポートします。他のエンコーディングのデータを使 用する場合は、UTF-8に変換してからErvizに渡してください。
  • シャープ記号 ("#") はコメントの開始を示します。シャープ記号が行の中に現れた場合、 そのシャープ記号から行末までの部分はコメントとみなされ、ERD画像の生成には影響を与えません。
  • Ervizでは以下の表記法をサポートします。
    • IE表記 :
      James Martin の Information Engineering 形式です。
    • IE-Strict表記 :
      IE形式とほぼ同じですが、エンティティとリレーションシップのスタイルが一部異なります。
    • IDEF1X表記 :
      IDEF1Xの仕様に従います。
  • 各表記法でのエンティティとリレーションシップのスタイルは以下のようになります。
    表記法 エンティティ リレーションシップ 説明
    独立 依存 非依存 依存 その他
    IE 角付 角丸 点線 実線 実線 IDEF1Xと同じスタイルです。
    IE-Strict 角付 角付 実線 実線 実線 常に角付の長方形と実線で描画します。
    IDEF1X 角付 角丸 点線 実線 実線 IDEF1Xの仕様に従います。
  • Ervizのテキストデータで使用する括弧には、以下のようなものがあります。
    名前 左 右
    角括弧 [ ]
    丸括弧 ( )
    山括弧 < >
    波括弧 { }
  • verb-reverseオプションは、動詞句の方向を修正するのに使用される。 
    #verb-reverseオプションはfalse (デフォルト)
#ER図での2番目の動詞句の方向が、テキストでの指定と異なる。
[E1] 1--* [E2] <from 1 to 2->
[E2] 1--* [E1] <from 2 to 1->



    #verb-reverseオプションはtrue
#verb-reverseオプションの働きにより、動詞句の方向が逆になる。
[E1] 1--* [E2] <from 1 to 2->
[E2] 1--* [E1] <from 2 to 1-> {verb-reverse: true}
